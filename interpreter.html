MIPS Interpreter

Implemented an instruction interpreter for a subset of MIPS code that will fetch, disassemble, decode and execute MIPS machine instructions to simulate each state in computation.

In this project, I create what is effectively a miniature version of MARS (MIPS Assembler and Runtime Simulator). The difference is MARS takes in assembly language source files and not .dump files.

Working over skeleton code, I complete the program by adding code to computer.c and simulate machine code versions of different MIPS machine instructions. In the skeleton code (framework code), it reads the machine code into memory, initializes the stack pointer, program counter, and all other registers. Then, it provides simulated data memory and stores instructions together with data in the same memory array. It sets flags that govern how the program interacts with the user and enters a loop that repeatedly fetches and executes instructions which prints information.

In the five processing stages of the MIPS architecture, I implement Decode, Execute, Mem, and RegWrite. Decode gives an instruction and performs register reads. Execute performs any ALU computation associated with the instruction and returns the value. Mem performs memory reads or writes related with the instruction. RegWrite performs any register writes that are needed. In my test code, I attempted to utilize mixes of all operational tasks to stack the fitting qualities into the input registers. The objective was to ensure that by utilizing the entirety of the directions with the right qualities, I could control the qualities to wind up letting out a similar outcome as the example test code did and in the correct format. I expected to ensure my code would work, so I included the instructions into the code to test the remainder of them.
