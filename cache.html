MIPS Cache Logic

Implemented a cache logic behind cache so that a MIPS simulator will be able to run MIPS instructions.

This project gave me intimate knowledge of cache logic through implementation of the actual caching logic. I used a MIPS simulator called TIPS(Thousands of Instructions Per Second) which to runs MIPS instructions but the cache logic is broken.

In this project, I implement the cache logic behind the cache so that TIPS can make use of many benefits caching entails. The initial TIPS code has default cache size of 0. I used a GUI that displays four main components such as register display, execution log, cache display, and control panel.

I completed the accessMemory() function in cachelogic.c which handles accessing actual memory. This makes the code in the accessMemory() function behave as a cache. To ensure a variety of caches can be simulated, I dynamically supported five properties: associativity (ranges from 1 to 5), number of unique indexes(2^n where n ranges from 0 to 4), block size(2^n bytes where n ranges from 2 to 5), replacement policy (LRU and Random), and memory synchronization policy (Write Back and Write Through).
